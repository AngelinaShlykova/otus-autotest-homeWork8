# Тест-план: API Wishlist

# Цель тестирования
Проверка корректности работы REST API для управления списками желаний: валидация статус-кодов, структуры ответов, обработки ошибок.

# Объекты тестирования

# Аутентификация
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/api/auth/register` | Регистрация пользователя |
| POST | `/api/auth/login` | Авторизация и получение токена |

# Списки желаний
| Метод | Эндпоинт | Описание | Требует авторизации |
|-------|----------|----------|---------------------|
| GET | `/api/wishlists` | Получить списки текущего пользователя |
| GET | `/api/wishlists/{id}` | Получить список по ID |
| POST | `/api/wishlists` | Создать новый список |
| PUT | `/api/wishlists/{id}` | Обновить список |
| DELETE | `/api/wishlists/{id}` | Удалить список |

# Подарки
| Метод | Эндпоинт | Описание | Требует авторизации |
|-------|----------|----------|---------------------|
| GET | `/api/gifts/wishlist/{wishListId}` | Получить подарки списка |
| POST | `/api/gifts/wishlist/{wishListId}` | Добавить подарок |
| GET | `/api/gifts/{id}` | Получить подарок по ID |
| PUT | `/api/gifts/{id}` | Обновить подарок |
| PUT | `/api/gifts/{id}/reserve` | Переключить статус резерва |
| DELETE | `/api/gifts/{id}` | Удалить подарок |

# Сценарии тестирования

# Позитивные тесты
1. Регистрация: Успешная регистрация с валидными данными - 200 OK
2. Логин: Вход с корректными учетными данными - 200 OK + JWT токен
3. CRUD WishList: Создание, чтение, обновление, удаление списка желаний
4. CRUD Gift: Добавление подарка в список, изменение цены, резервирование
5. Валидация полей: Проверка, что `title` у списка и `name`+`price` у подарка обязательны

# Негативные тесты
1. Регистрация:
   - Пустой `username` (< 3 символов) - 400 Bad Request
   - Короткий `password` (< 6 символов) - 400
   - Неверный формат `email` - 400
   - Регистрация с уже существующим email - 409 Conflict
2. Авторизация:
   - Неверный пароль - 401 Unauthorized
   - Пользователь не найден - 404
3. Доступ к ресурсам:
   - Запрос к защищенному эндпоинту без токена - 401/403
   - Запрос с невалидным/просроченным токеном - 401
4. Валидация данных:
   - Создание подарка без обязательного поля `name` - 400
   - Цена подарка < 0 - 400
   - `storeUrl` с неверным форматом - 400
5. Работа с несуществующими ресурсами:
   - Получение/удаление подарка по несуществующему UUID - 404
